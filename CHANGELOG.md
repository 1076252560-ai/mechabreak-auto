# Changelog

## v1.0.6 (2025-06-02)

- `arm_thresholds` 配置项：可为任意武装单独设置匹配阈值
- 蓄能爆破炮默认阈值 0.92，其他武装 0.90
- GUI 控件变化实时保存到 config.json（500ms 防抖）

## v1.0.5 (2025-06-02)

- IV 级选择改为 3 个互斥模式：购买所有 / 仅购买勾选武装 / 不购买
- 武装选择 (I/II/III 级) 与 IV 级选择分开为两个区域
- 修复 IV 级"仅购买勾选武装"模式下 1/2/3 级卡片被错误跳过
- IV 级判断优先于 1/2/3 级

## v1.0.4 (2025-06-02)

- ARM_THRESHOLD 提高到 0.90，修复蓄能射束炮误识别为蓄能爆破炮
- 卡片检测仅限框选区域内 3x2 等分
- 未配卡片/刷新区域时禁用开始，GUI 红字提示
- 刷新按钮仅用框选坐标，删除模板匹配

## v1.0.3 (2025-06-02)

- EXE 文件名带版本号
- 打包加入 `--uac-admin` 自动管理员提权

## v1.0.2 (2025-06-02)

- 刷新按钮优先使用框选坐标
- 更新默认配置为 v1.0.1

## v1.0.1 (2025-06-02)

- F8 全局热键停止
- `run.bat` 本地一键运行
- 默认配置使用用户实际设置

## v1.0.0 (2025-05-27)

- 模板匹配识别武装和已售状态
- 紫色检测 IV 级武装
- Win32 API 模拟键鼠（SendInput + keybd_event）
- tkinter GUI 置顶窗口
- 全选 / IV级 / 按名称三种筛选模式
- 设置自动保存，下次启动恢复
- GitHub Actions 自动打包 Release
